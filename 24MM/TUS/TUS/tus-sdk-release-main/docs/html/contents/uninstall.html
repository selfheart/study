3.1.1. Uninstall SDK 03X¶

This procedure explains how to uninstall SDK030.

3.1.1.1. Uninstall files on Target Device¶

On the Target Device, it is needed to remove some deployed files and configuration files. If you follow tutorials, please

  • remove folder defined by ${TARGET_DEPLOY_DIR} on the tutorial page.

  • for only EMOOTA SDP licensed user, additionally removing /data folder is needed.

3.1.1.2. Uninstall cross compile environments¶

Please execute below commands after stopping containers if they are still running.

docker rmi tus-cross-yocto40:latest
docker rmi tus-cross-build:latest

3.1.1.3. Uninstall Host Tools¶

Please execute below commands on the SDK root directory.

docker rmi tus-docker-registry/sdk:$(cat TUS_SDK_VERSION)

In addition to that, if you build Host Tools

docker rmi tus-tool-build:$(cat TUS_SDK_VERSION)

And if ${TUS_SDK}/bin/ is added to $PATH, please update $PATH to remove it. Additionally please unset ${TUS_SDK} environment variable.

After completing above uninstall steps, it is able to remove deliverable archives tus-XXX.txz ( XXX is equal to the version number like 020 ) and extracted folders.

This procedure explains how to uninstall SDK020, SDK021 and SDK022.

3.1.2.1. Uninstall files on Target Device¶

On the Target Device, it is needed to remove some deployed files and configuration files. If you follow tutorials, please

  • remove folder defined by ${TARGET_BUILD_DIR} on the tutorial page.

  • for only EMOOTA SDP licensed user, additionally removing /data folder is needed.

3.1.2.2. Uninstall cross compile environments¶

Please execute below commands after stopping containers if they are still running.

docker rmi tus-cross-yocto40:latest
docker rmi tus-cross-build:latest

3.1.2.3. Uninstall Host Tools¶

Please execute below commands on the SDK root directory.

docker rmi tus/sdk:$(cat TUS_SDK_VERSION)

And if ${TUS_SDK}/bin/ is added to $PATH, please update $PATH to remove it. Additionally please unset ${TUS_SDK} environment variable.

After completing above uninstall steps, it is able to remove deliverable archives tus-XXX.txz ( XXX is equal to the version number like 020 ) and extracted folders.

3.1.3. Uninstall SDK 010¶

3.1.3.1. Uninstall files on Target Device¶

On the Target Device, it is needed to remove some deployed files and configuration files.

If you follow tutorials, please

  • remove folder defined by ${TARGET_DEPLOY_DIR} on the tutorial page.

  • remove /var/www/html/tus on Target Device where Update Orchestrator runs.

  • for only EMOOTA SDP licensed user, additionally removing /data folder is needed.

3.1.3.2. Uninstall cross compile environments¶

First, please remove cross compiled libraries.

sudo rm -rf /usr/aarch64-linux-gnu/include/openssl
sudo rm -rf /usr/aarch64-linux-gnu/include/curl

pushd ${TUS_SDK}/src/lib/

pushd ./grpc/cmake/build_aarch64
cat install_manifest.txt | sudo xargs rm -rf
popd 

pushd ./grpc/cmake/build
cat install_manifest.txt | sudo xargs rm -rf
popd 

pushd ./protobuf
sudo make uninstall
popd

pushd ./lua-5.4.3
sudo make INSTALL_TOP=" /usr/aarch64-linux-gnu" uninstall
popd 

popd

And then please remove libraries installed by package managers.

sudo pip uninstall -y protobuf==3.12.2
sudo pip uninstall -y ecdsa==0.17 lz4==4.0.0 cryptography==36.0.1 mypy==0.931 flake8==4.0.1 Flask==2.1.1 requests==2.26.0
sudo apt-get remove -y ca-certificates \
    autoconf \
    automake \
    cmake \
    curl \
    g++-aarch64-linux-gnu \
    g++ \
    gcc-aarch64-linux-gnu \
    gcc \
    git \
    libcurl4-openssl-dev:arm64 \
    liblz4-dev:arm64 \
    libssl-dev \
    libssl-dev:arm64 \
    libtool \
    make \
    ninja-build \
    openssl \
    python3 \
    python3-distutils \
    python3-pip \
    unzip 
sudo apt-get remove -y `dpkg --get-selections | grep arm64 | awk '{print $1}'`
sudo apt autoremove -y

Finally please revert sources.list.

sudo dpkg --remove-architecture arm64
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal main restricted/d" /etc/apt/sources.list
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal-updates main restricted/d" /etc/apt/sources.list
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal universe/d" /etc/apt/sources.list
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal-updates universe/d" /etc/apt/sources.list
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal multiverse/d" /etc/apt/sources.list
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal-updates multiverse/d" /etc/apt/sources.list
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal-backports main restricted universe multiverse/d" /etc/apt/sources.list
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal-security main restricted/d" /etc/apt/sources.list
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal-security universe/d" /etc/apt/sources.list
sudo sed -i "/deb \[arch=arm64\] http:\/\/ports.ubuntu.com\/ubuntu-ports focal-security multiverse/d" /etc/apt/sources.list

sudo sed -i "s/deb \[arch=amd64\] /deb /g" /etc/apt/sources.list

3.1.3.3. Uninstall Host Tools¶

Please execute below commands.

docker rmi tus/sdk:0.1

And if ${TUS_SDK}/deliverables/bin/ is added to $PATH, please update $PATH to remove it. Additionally please unset ${TUS_SDK} environment variable.

After completing above uninstall steps, it is able to remove deliverable archives tus-010.txz and extracted folders.
