4.3.1. EMOOTA SDP v3.0 Subcommand¶

EMOOTA SDP v3.0 subcommand usage is following:

$ tus pdp sdpv3 -h
usage: tus pdp sdpv3 [-h] [--endpoint ENDPOINT] [--username USERNAME]
                     {rest,ul-pkg,get-pkg-status,get-vconf-info,approve-pkg,dl-pkg}
                     ...

EMOOTA SDPv3 operator To set username and password of EMOOTA SDPv3, please
define SDP_USERNAME and SDP_PASSWD environment variables.

positional arguments:
  {rest,ul-pkg,get-pkg-status,get-vconf-info,approve-pkg,dl-pkg}
                        help
    rest                wraps some REST APIs
    ul-pkg              upload package files including TUP to pdp.
    get-pkg-status      check status of uploaded file on pdp.
    get-vconf-info      get current vehicle configuration information
    approve-pkg         approve uploaded file on pdp.
    dl-pkg              download package files including TUP

optional arguments:
  -h, --help            show this help message and exit
  --endpoint ENDPOINT   base URL of EMOOTA SDPv3.
                        default='https://t-poc-r3-ap-
                        northeast-1-b2b.t-poc-r3.com/'
  --username USERNAME   username of EMOOTA SDPv3. SDP_USERNAME environment
                        variable is used if not specified. SDP_PASSWD
                        environment variable should be defined.

  • To use this subcommand, EMOOTA SDP v3.0 user account is needed.

  • EMOOTA SDP v3.0 command prints error information including EMOOTA SDP v3.0 error code if HTTP Status code is not 20x. To know EMOOTA SDP v3.0 error code details, please refer to EMOOTA OTAmatic documents.

4.3.1.1. Common operations¶

4.3.1.1.1. upload packages including TUP and wait for upload to be processed¶

$ tus pdp sdpv3 ul-pkg -h
usage: tus pdp sdpv3 ul-pkg [-h]
                            [--file-type {VSPA,REPRO_METADATA,DOWNLOAD_METADATA}]
                            [--file_hash_value FILE_HASH_VALUE]
                            [--target_id TARGET_ID] [--part_size PART_SIZE]
                            [--approve] [--async_ops]
                            id file-path

positional arguments:
  id                    change event package id
  file-path             package file path, if VSPA is passed this file is
                        archived by ZIP format automatically.

optional arguments:
  -h, --help            show this help message and exit
  --file-type {VSPA,REPRO_METADATA,DOWNLOAD_METADATA}
                        file type (default: VSPA)
  --file_hash_value FILE_HASH_VALUE
                        Hash Values of the file. This is mandatory for
                        specific file types like REPRO_UPDATE, ROLLBACK_DATA
                        etc. (default: None)
  --target_id TARGET_ID
                        target id. This is mandatory for specific file types
                        like REPRO_UPDATE, ROLLBACK_DATA etc. (default: None)
  --part_size PART_SIZE
                        Maximum size of a content to upload on a each single
                        upload. Should be within 5242880 and 5368709120 byte
                        (default: 5242880)
  --approve             if this option is passed, after uploading package this
                        automatically approves. (default: False)
  --async_ops           if this option is passed, it will upload packages
                        without checking upload status on pdp. if --approve is
                        set as well, it will wait until uploaded package
                        status becomes approvable. (default: False)

Before uploading, creating change event package by tus pdp sdpv3 rest create-chevent is needed.

For EMOOTA SDP v3.0, at least 3 files must be uploaded.

  • VSPA can be used for TUP. This operator creates zip file automatically and upload it because EMOOTA SDP v3.0 requires zip file for VSPA.

  • DOWNLOAD_METADATA can be used for metadata. This shall include just filename of TUP as follows.

$ cat dl_meta.txt
tutorial_outer.tup

  • REPRO_METADATA can be used for metadata. Target Device does not read this file, but writing some dummy text content is needed for uploading.
