1. Document Architecture¶

The following documents are provided for Trusted Update System and Trusted Update Package.

1.1. Trusted Update System Documents¶

Document Purpose
Trusted Update System Overview Explains the overall TUS(=Trusted Update System). You can learn key concepts of TUS (e.g. Update Orchestrator, Domain Controller, etc.) with the document. There are also feature list and quality criteria of the TUS written in the document.
TUS SDK Roadmap Shows the strategic plan of the TUS SDK development. You can refer to the latest information and change history of the roadmap.
TUS SDK Structure Overview (In an Appendix) There are multiple materials used for the TUS demonstrations to explain the TUS SDK structure. Please refer to these as an aid to better understand the TUS and the TUS SDK.

1.1.1. SDK Documents¶

Document Purpose
Tutorials Explains step-by-step instruction on how to get started with the TUS SDK. For example, you can learn how to install Update Orchestrator on Target Device using the Host Tools.
TUS CLI Reference Provides the TUS Command Line Interface reference.
API Reference Provides the TUS SDK's Lua API reference for operations via Lua script.
Release Note Shows detail changes included in the current version of SDK.

1.2. Trusted Update Package Documents¶

Document Purpose
Trusted Update Package Introduction Explains briefly the abstract of TUP(=Trusted Update Package). After checking this document, refer to Trusted Update Package Design for further details.
Trusted Update Package Format Design Explains the design of TUP file format and its rationale. After checking this document, refer to Trusted Update Package Format Specification for detail file format specification.
Trusted Update Package Format Specification Explains the structure inside the TUP. Please refer to this document to learn the detail file format of TUP.
